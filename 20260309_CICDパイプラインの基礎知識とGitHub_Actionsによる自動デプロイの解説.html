### 現在の開発フロー：ローカル→GitHub→Google Cloud

青木は、受講者が現時点で構築している開発環境の全体像を整理した。具体的には以下の3層構造になっている：

### CI/CDパイプラインの仕組みと自動化の流れ

青木は、CI/CDパイプラインをつなぐことで何が変わるかを具体的なフローで説明した。

**パイプラインをつないだ場合の自動フロー：**

1. ローカルのCursorで作業・修正を行う

2. コミット・プッシュを実行すると、GitHubにコードが保存される

3. GitHubに上がったコードを受けて、**GitHub Actions**が自動起動する

4. GitHub Actionsがオンライン上でテスト・動作確認を自動実行する

5. 問題がなければ、そのまま自動的にGoogle Cloud（本番環境）に反映される

通常は「コミット・プッシュ → GitHub保存」で止まり、本番環境への反映には別途手動操作が必要だが、パイプライン構築後はコミット・プッシュだけで本番反映まで完結する。

**GitHub Actionsの役割**については、GitHubの機能の一つとして紹介され、「オンライン上でテストして問題ないかを自動チェックする」ことが主な役割として説明された。本番環境に上げる前のバリデーション層として機能する。
